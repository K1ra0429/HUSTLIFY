# Отдельная веб-админка (как отдельный сайт)

Один статический HTML-файл, без сборки. Управляет разделом «Наши кейсы»
на главной странице через ту же edge-функцию `admin-api` и ту же базу
Supabase, что и основной сайт — то есть правки сразу видны на сайте.

## Настройка (один раз, перед деплоем)

Открой `index.html`, в начале `<script>` уже стоит правильный `SUPABASE_URL`
проекта. Нужно только вписать реальный анонимный ключ:

```js
const SUPABASE_URL = "https://unjcwqmgthsvoyoeaxdd.supabase.co";
const SUPABASE_ANON_KEY = "PASTE_REAL_ANON_KEY_HERE"; // <-- сюда
```

Ключ взять там же, где брали для основного сайта:
Supabase Dashboard → проект → Settings → API → `anon` / `public` key
(это тот же ключ, что стоит в `.env` как `VITE_SUPABASE_PUBLISHABLE_KEY`).

## Деплой

Обычный статический файл — заливается на любой хостинг для статики
(Vercel/Netlify как отдельный проект с этой папкой в качестве корня,
GitHub Pages, Cloudflare Pages и т.п.). Сборка не нужна.

## Вход

Логин и пароль выдаёт бот при добавлении модератора (раздел
«🛡 Модераторы» в `/admin` бота, доступен только тем, чей Telegram ID
прописан в `ADMIN_TELEGRAM_IDS`).

Первый (главный) вход по умолчанию:
- Логин: `admin`
- Пароль: `123`

**Обязательно смени этот пароль** — добавь через бота нового модератора
и не используй дефолтный аккаунт в проде.

## Что нужно накатить в Supabase перед первым использованием

Одна новая миграция (`supabase/migrations/20260705090000_moderators_web_panel.sql`)
добавляет две новые таблицы: `moderators` и `admin_web_sessions`.
Она НЕ трогает существующую таблицу `cases` — данные, которые уже
есть на сайте, останутся как есть.

```bash
supabase db push
```

Затем передеплой двух функций, чтобы подхватили новый код:

```bash
supabase functions deploy admin-api --no-verify-jwt
supabase functions deploy telegram-bot --no-verify-jwt
```

## Как это работает технически

- Все данные и авторизация идут через ту же edge-функцию `admin-api`,
  что уже работает у React-панели (`/admin` внутри основного сайта) —
  никакого дублирования бэкенда, только новый способ входа.
- Вход — не Supabase Auth, а свой логин/пароль (SHA-256 хэш в таблице
  `moderators`) и свой сессионный токен (таблица `admin_web_sessions`,
  живёт 7 дней, хранится в `sessionStorage` браузера).
- Добавлять/удалять модераторов можно только из Telegram-бота
  («🛡 Модераторы» в `/admin`, доступно только супер-админам из
  `ADMIN_TELEGRAM_IDS`).
- React-панель `/admin` внутри сайта (вход по общему `ADMIN_PASSWORD`)
  продолжает работать как раньше — эта отдельная страница ей не мешает,
  это просто ещё один способ попасть в ту же базу.
